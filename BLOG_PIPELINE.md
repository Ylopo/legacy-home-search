**Market:** Virginia Beach, Chesapeake, Norfolk, Suffolk, Hampton, Newport News (Hampton Roads, VA)

```

PATH A — Idea Review (primary)
─────────────────────────────────────────────────────────

Daily / Weekly Research (Cron)
  └─ Tavily: 8–10 article searches (3 pinned + 5–7 rotating)
  └─ Renick Blog analytics scrape (weekly) — pattern extraction
  └─ Required evergreen topics coverage audit (monthly)
  └─ All scored → stored as ideas in Redis (lhs:ideas:queue)

Weekly Idea Digest (Cron — Thu/Fri 8 AM PT)
  └─ Top ideas emailed to kiwi@ylopo.com
  └─ Link to /admin/idea-review?secret=…

Operator approves an idea on /admin/idea-review
  └─ /api/content/ideas/approve (POST)
       └─ Claude Sonnet writes full blog post
       └─ Sanity doc created: workflowStatus: 'media_pending'
       └─ Page redirects to /admin/va-queue automatically

─────────────────────────────────────────────────────────
PATH B — Blog Picker (fast track for daily news)
─────────────────────────────────────────────────────────

Operator visits /admin/blog-picker/[date]?secret=…
  └─ Shows articles scored by daily research cron (from Redis)
  └─ Operator selects 1–5 articles and clicks "Write & Queue"
  └─ /api/blog/publish (POST)
       └─ Claude Sonnet writes full blog post for each selected article
       └─ Sanity doc created: workflowStatus: 'media_pending'
       └─ Success screen shows "Queued" + link to /admin/va-queue

─────────────────────────────────────────────────────────
BOTH PATHS CONVERGE HERE → VA Queue
─────────────────────────────────────────────────────────

VA opens /admin/va-queue
  └─ "Needs Media" section: all media_pending posts
  └─ "Ready to Publish" section: all media_ready posts
  └─ Trash button on any card → delete duplicate/unwanted posts

VA opens a post → /admin/va-queue/[postId]
  └─ 3-step nav visible: Idea Review → Media Queue → Post Editor

  STEP 1 — Thumbnail
    └─ AI generate: prompt editor + background picker → /api/blog/generate-thumbnail
    └─ OR upload an image manually (Vercel Blob client upload)
    └─ Preview renders inline

  STEP 2 — Social Copy
    └─ Auto-generated by Claude Haiku (editable)
    └─ Used for Facebook caption and video descriptions

  STEP 3 — Video (optional)
    └─ Generate base video: HeyGen avatar → poll until ready → download link
    └─ Edit the downloaded video externally (add B-roll, captions, etc.)
    └─ Upload final video → /api/content/upload-video (Vercel Blob, random suffix)
    └─ Optional: upload a video thumbnail image separately

  STEP 4 — Publish
    └─ "🚀 Publish" → /api/content/publish
         └─ If thumbnail was just uploaded: saves coverImage to Sanity first (inline mark-ready)
         └─ Sets workflowStatus: 'published' + publishedAt: now (website live)
         └─ Posts to Facebook + LinkedIn + X via Blotato (each with unique platform-specific caption)
         └─ If videoUrl present: also publishes to YouTube + TikTok + Facebook Reel; LinkedIn/X get video-first, fall back to image if rejected
         └─ Polls Blotato → stores all submission IDs + post URLs on doc
```

## Required Evergreen Topics (Coverage System)

A registry of high-performing post templates that **must be covered** for every Hampton Roads city the client serves. The system audits Sanity, identifies gaps, and seeds them as IdeaCandidates so they appear on `/admin/idea-review` for approval — without ever duplicating a post that already exists.

### Required topic templates

| Template | Cities | Audience | Category |
|----------|--------|----------|----------|
| `What Does It Cost to Buy a Home in [City] in 2026?` | Hampton Roads + 6 cities | Buyer | cost-breakdown |
| `What Does It Cost to Sell a Home in [City]?` | Hampton Roads + 6 cities | Seller | cost-breakdown |
| `How Do Virginia Property Taxes Work for [City] Home Buyers?` | Hampton Roads + 6 cities | Buyer | buying-tips |
| `[City A] vs [City B]: Which Is Better for [Buyers/Investors] in 2026?` | 5 priority pairs × 2 audiences | Buyer/Investor | community-spotlight |
| `Is 2026 a Good Time to Buy in [City]?` | Hampton Roads + 6 cities | Buyer | buying-tips |
| `What Happens After Your Offer Is Accepted in Virginia?` | State-level (1 post) | Buyer | buying-tips |
| `What Do Flood Zones Mean for Home Buyers in [City]?` | Coastal cities (5 cities) | Buyer | flood-and-risk |

**44 total topic goals** across all templates and city/audience combinations.

### How coverage detection works

`lib/required-topics.ts` defines each goal with `matchAllOf: string[]` — a list of substrings that must ALL appear in a published post's title or slug for the goal to be considered covered. Disambiguation is built in (e.g., a "Hampton Roads" post does not falsely match a "Hampton" city goal).

### How gap seeding works

`/api/cron/required-topics-coverage` runs on the **3rd of each month at 9 AM PT** (also POST-triggerable manually). For each goal:
- ✅ **Covered by published post** → skip
- ⏳ **Already pending in idea queue** → skip
- 🆕 **Missing AND not queued** → seed as `IdeaCandidate` with `source: 'internal'`, score ~70, marked `Required Evergreen`

### Manual operations

```bash
# Dry run — preview gaps without seeding
curl -X POST "https://www.legacyhometeamlpt.com/api/cron/required-topics-coverage?secret=ADMIN_SECRET&dryRun=1"

# Run — seed any missing goals into the idea queue
curl -X POST "https://www.legacyhometeamlpt.com/api/cron/required-topics-coverage?secret=ADMIN_SECRET"
```

### Adding new required topic templates

Edit `lib/required-topics.ts` → append a generator to `REQUIRED_TOPICS`. The next cron run picks up the new goals and seeds gaps. Existing posts and queued ideas are unaffected.

### Files

| File | Purpose |
|------|---------|
| `lib/required-topics.ts` | Registry: defines all topic templates and city expansions |
| `lib/required-topics-coverage.ts` | `checkCoverage()` + `seedMissingTopics()` |
| `app/api/cron/required-topics-coverage/route.ts` | GET (cron) and POST (manual) endpoint |

## Phase 1: Research & Idea Engine

### Research Cron (`/api/cron/research`)

Runs daily at 6 AM PT. Searches Tavily for Hampton Roads real estate news, scores results against the 100-point model, and stores passing ideas in Redis.

**Pinned daily queries (always run):**
```
'Virginia Beach real estate market news 2026'
'Hampton Roads housing market trends 2026'
'Hampton Roads community development news projects 2026'
```

**End-of-month event queries (days 22–31 only, pinned alongside the above):**
```
'Virginia Beach Norfolk Hampton Roads events festivals things to do next month'
'Hampton Roads community events attractions activities upcoming month'
'Virginia Beach Norfolk Chesapeake local events concerts festivals calendar'
```

During the last 10 days of each month these 3 queries are added automatically so the idea queue surfaces upcoming events and attractions for the following month — giving the audience a heads-up they can act on. See [End-of-Month Events Window](#end-of-month-events-window) below.

**Rotating query pool (47 topics, 5 slots/day cycle through):**
- Virginia Beach home prices / Chesapeake / Norfolk / Suffolk / Newport News / Hampton real estate
- Hampton Roads rental market / investment returns / short-term rental regulations / Airbnb
- Virginia HOA law / property tax / zoning / landlord-tenant law
- Hampton Roads major development / Port of Virginia / resort area projects / downtown arena
- Military: PCS season / base expansion / VA loans / BAH rates / NAS Oceana / Norfolk Naval Station
- Virginia Beach oceanfront / waterfront / new construction neighborhoods
- First-time buyer programs / VHDA grants / down payment assistance
- Flood insurance / FEMA flood zone / sea level rise / elevation certificates
- Homeowner insurance rates / wind and storm coverage in coastal Virginia
- Cost of ownership breakdown / closing costs / property tax comparison across Hampton Roads cities
- Hampton Roads local events / festivals / economic activity signals
- New business openings / corporate relocations signaling housing demand
- 55+ retirement communities / senior housing in Hampton Roads
- Schools and school district ratings for family buyers
- Rent vs. buy analysis for Hampton Roads market

### Renick Pipeline (`/api/cron/renick-pipeline`)

Scrapes the Renick Blog Analytics dashboard via Tavily HTML extraction (no API). Extracts top-performing post patterns (title formats, categories, CTR lift). Converts patterns into idea candidates scored by the 100-point model. Feeds into the same Redis idea queue — no separate pipeline.

### Idea Scoring Model (100 points)

| Dimension | Max | Description |
|-----------|-----|-------------|
| Local Relevance | 25 | Hampton Roads / Virginia-specific, not national filler |
| Timeliness | 20 | Time-sensitive or evergreen with fresh angle |
| Format Fit | 15 | Matches proven formats (cost breakdown, Q&A, how-to, comparison) |
| Audience Value | 15 | Directly useful to Hampton Roads buyers and homeowners |
| Source Credibility | 10 | Government data, local publication, established real estate source |
| Novelty | 10 | Not a repeat of recent content; new angle |
| SEO Potential | 5 | Matches how people actually search |
| **End-of-Month Event Boost** | **+15** | Applied to `community-spotlight` and `news` ideas on days 25–31; +8 on days 22–24. Automatically surfaces event/attractions content during the last week of every month. |

**Pass threshold:** ≥55/100 stored in Redis. Ideas scoring ≥85 with `timeliness: 'breaking'` flag trigger an immediate urgent email to kiwi@ylopo.com, bypassing the weekly queue.

### End-of-Month Events Window

**What it is:** During the last 10 days of every month (days 22–31), the content pipeline automatically shifts priority toward community events, festivals, and local attractions happening in the upcoming month.

**How it works:**
1. **Extra research queries** — 3 event-focused Tavily searches are added to the daily pinned set (days 22–31 only), so the raw article pool includes upcoming-event content alongside normal real estate news
2. **Scoring boost** — Ideas categorized as `community-spotlight` or `news` receive a deterministic bonus applied after all other dimensions: +15 points on days 25–31, +8 points on days 22–24
3. **Events cron still runs** — On the 25th of each month, `/api/cron/events-research` runs a dedicated search for next-month events and writes posts directly to the VA queue — the scoring boost makes sure those ideas also win if they come through the daily research path instead

**What operators see:** In the idea queue during the last week of each month, community/events ideas will appear at or near the top even when competing with real estate news. The operator picks them as normal — the scoring ensures they're surfaced, but the human still decides what gets written.

**Categories that receive the boost:** `community-spotlight`, `news`

### Redis Idea Store

Key: `lhs:ideas:queue` (sorted set, score = idea score desc)
TTL: 7 days
Each entry: `{ id, title, url, excerpt, category, score, breakdown, source, timeliness, savedAt }`

## Phase 2: Idea Review

### `/admin/idea-review`

The operator reviews scored ideas, approves ones worth writing, and skips/defers others. This is the only gate before a post gets written — Claude writes nothing until an idea is approved here.

**Actions per idea:**
- **Approve & Write Post** → writes post immediately (see Phase 3), then **auto-redirects to `/admin/va-queue`**
- **Defer** → keeps in queue for next week's digest
- **Skip** → removes from queue permanently

**Score rings:** Click a score ring on any idea card to expand the 7-dimension score breakdown.

**"Generate More Ideas" button:** Manually triggers `/api/content/ideas/generate` — runs `fetchAndScoreIdeas()` on demand and stores results without waiting for the daily cron.

### Idea Digest Email (Weekly)

Sent Thursday or Friday at 8 AM PT to `kiwi@ylopo.com`. Lists top 10–15 scored ideas with title, score, category, source, and a direct link to `/admin/idea-review`. The email replaces the old daily digest and the old Renick approval email.

## Phase 3: Post Writing

### `/api/content/ideas/approve` and `/api/blog/publish`

Both paths call the same underlying writing function. Claude Sonnet 4.6 writes a full blog post from the idea's title, excerpt, and URL. Post is saved to Sanity CMS with `workflowStatus: 'media_pending'` — hidden from the public blog until the VA adds a thumbnail and publishes.

**`publishedAt` behavior:** Set to the actual publish time (when the VA clicks "Publish"), not the date the post was written. This ensures the date shown on the blog reflects when the article went live.

**Post writing rules:**
- Voice: Virginia Beach real estate professional at Legacy Home Search — knowledgeable, approachable, never salesy
- Ties national/Virginia news back to the local Hampton Roads market
- Structure: engaging headline → intro → 2–3 body sections (`##` headings) → `## What This Means For You` (3–4 bullets) → closing → source credit (linked)
- All posts must answer: "What does this mean for Hampton Roads buyers and homeowners?"

### VA Tips & Troubleshooting

**If Publish fails:** `workflowStatus` resets to `publish_failed` and the error is shown inline. Click Publish again to retry. If it keeps failing, check the Blotato connection or contact the admin.

**If a generated thumbnail looks wrong:** Edit the prompt and click Generate again — you can regenerate as many times as needed before publishing. The more specific the prompt, the better the result. YouTube-style thumbnails work best: bold text, high contrast, simple composition.

**If a post isn't in the queue:** It's either already published or hasn't been approved yet. Check with the admin.

**Uploading an external thumbnail:** You can upload an image made in Canva, Photoshop, or any other tool instead of generating one — just use the Upload Image button.

**Caption review:** The social copy textarea shows the Facebook caption. At publish time, Claude Haiku generates unique variations for all 5 platforms (Facebook, LinkedIn, X, TikTok, YouTube) from the same article. The blog post URL is appended automatically to each. If you edit the caption textarea, your custom text is used as the base for all platforms.

## Phase 5: Publishing

### Website + Facebook (main publish)

When the VA clicks **Publish**:

1. `workflowStatus` set to `'publishing'`
2. Social copy finalized (VA-edited or auto-generated by Claude Haiku)
3. Sanity image URL resolved at 1200px width
4. Blotato `POST /v2/posts` called with `accountId`, `pageId`, `text + url`, `imageUrl`
5. `blotatoPostSubmissionId` stored on Sanity doc
6. `workflowStatus` set to `'published'` + `publishedAt` set to current timestamp (website live)
7. Client polls `/api/content/blotato-status` every 10s (up to 30 attempts / ~5 minutes)
8. On resolution: `blotatoPublishStatus`, `blotatoPublishedAt`, `facebookPostUrl` stored on doc

### YouTube + TikTok (video publish)

After the post is published to website/Facebook and a video has been uploaded, the VA can click **▶ Publish Video to YouTube & TikTok**:

1. Calls `/api/content/publish-video` (POST)
2. Saves `videoUrl` (and `videoThumbnailUrl` if set) to Sanity doc
3. YouTube and TikTok publish via `Promise.allSettled` — one failure does not block the other
4. `youtubePostSubmissionId` and `tiktokPostSubmissionId` stored on doc
5. Per-platform result shown inline (success or error message)

**YouTube error — account verification:** If YouTube returns a 422 about "custom thumbnails", the YouTube channel needs to be verified at youtube.com/features and reconnected in Blotato. This is a one-time setup step.

### Blotato API

All platforms use `POST /v2/posts` with platform-specific `accountId` and `targetType`.

```
# Account IDs
BLOTATO_FACEBOOK_ACCOUNT_ID = 29353   (platform: "facebook", targetType: "facebook")
BLOTATO_YOUTUBE_ACCOUNT_ID  = 34912   (platform: "youtube",  targetType: "youtube")
BLOTATO_TIKTOK_ACCOUNT_ID   = 39905   (platform: "tiktok",   targetType: "tiktok")
BLOTATO_LINKEDIN_ACCOUNT_ID = 20512   (platform: "linkedin", targetType: "linkedin")
BLOTATO_X_ACCOUNT_ID        = 17792   (platform: "twitter",  targetType: "twitter")
BLOTATO_FACEBOOK_PAGE_ID    = 1101893253009079

# Caption character limits enforced in blotato-client.ts:
#   Facebook:  2000 chars
#   LinkedIn:  3000 chars
#   X/Twitter: 257 chars (leaves room for appended URL ~23 chars)
#   YouTube:   no limit enforced
#   TikTok:    no limit enforced

# Poll status
GET /v2/posts/{postSubmissionId}
→ { status: "published" | "pending" | "failed", postUrl? }
```

### Workflow Status Model

| Status | Visible on website? | Meaning |
|--------|--------------------|----|
| `media_pending` | No | Post written; VA needs to add thumbnail |
| `media_ready` | No | Thumbnail saved; ready to publish |
| `publish_pending` | No | Publish triggered, not yet confirmed |
| `publishing` | No | Blotato call in progress |
| `published` | Yes | Live on website + Facebook |
| `publish_failed` | No | Blotato failed; VA can retry |

**Legacy compatibility:** Old posts with `status: 'published'` remain live. GROQ filter: `workflowStatus == 'published' || status == 'published'`.

**Failure handling:** If Blotato returns an error, `workflowStatus` is set to `publish_failed` and the error is shown inline in the VA editor. The VA can retry by clicking Publish again on the same post.

**Multi-client (future):** Phase 1 uses hardcoded account/page IDs from env vars. Phase 2 will replace these with a Sanity `clientConfig` document per client. The getter functions in `lib/blotato-client.ts` are the only files to update (`getFacebookAccountId`, `getYouTubeAccountId`, `getTikTokAccountId`, `getLinkedInAccountId`, `getXAccountId`, `getPageId`).

## Phase 6: Monthly Progress Tracker

The bottom of the VA Queue page shows a month-to-date progress panel tracking output against daily goals.

**Goals:**
- Blog posts (text only, no video): **2/day** → 60/month
- Video posts (post with `videoUrl` set): **1/day** → 30/month

**Logic:** `GET /api/content/monthly-stats` queries all published posts since the 1st of the current month and returns `{ totalPosts, videoPosts }`. Every published post counts toward the blog total (video posts are double-counted — they satisfy both goals). Video posts = published with `videoUrl`.

**Status thresholds** (compared to expected pace for the current day of month):
- **On Track** — actual ≥ expected
- **Slightly Behind** — actual ≥ 75% of expected
- **Behind** — actual < 75% of expected

Color coding: green / amber / red matching the thresholds above.

## Phase 7: Content Refresh

### Philosophy

New posts should mature before being touched. The refresh system evaluates all published posts on a cadence, scores them by urgency, and surfaces the highest-priority candidates for operator review. Approved posts are rewritten in-place by Claude — they stay live and update immediately via ISR.

**The rule:** Don't touch posts younger than 90 days. After that, each post gets reviewed on its assigned tier's cadence. Performance triggers (once GA4 is wired up) will accelerate candidates that are declining.

### Refresh Tiers

| Tier | First Review | Subsequent | Auto-assigned from |
|------|-------------|-----------|-------------------|
| `fast-changing` | 30 days | 90 days | market-update, cost-breakdown |
| `news-trend` | 30 days | 90 days | news |
| `competitive` | 90 days | 180 days | buying-tips, selling-tips, investment, flood-and-risk |
| `money-page` | 60 days | 120 days | (manually set in Sanity Studio) |
| `pillar` | 90 days | 180 days | (manually set) |
| `seasonal` | 60 days before peak | 365 days | (manually set) |
| `evergreen` | 120 days | 365 days | community-spotlight |

Tier can be overridden per post in Sanity Studio (`Refresh Tier` field). If blank, the system auto-assigns from category.

### Priority Scoring (0–100)

| Component | Max | Logic |
|-----------|-----|-------|
| Overdue factor | 30 | 0 if not yet due; scales 0→30 as overdue increases |
| Topic volatility | 20 | Based on tier — fast-changing=18, news-trend=20, competitive=15 |
| Category importance | 15 | cost-breakdown=15, buying-tips=13, investment/flood-and-risk=11 |
| Age score | 15 | <90d=0, 90-180d=5, 180-365d=10, >365d=15 |
| Seasonal window | 20 | Bonus when season peak is within 60 days |

**Action thresholds:** ≥ 75 → Full Refresh · ≥ 45 → Light Refresh · ≥ 20 → Review Only · < 20 → Do Not Touch

### Admin Workflow

1. **Monday 10 AM PT** — cron runs `/api/cron/refresh-evaluation` → evaluates all published posts, builds Redis queue, sends email digest to operator
2. **Operator reviews email** → clicks "Review Refresh Queue →"
3. **Visit `/admin/refresh-queue?secret=…`** → shows scored candidates with tier, age, score, reasons, playbook
4. For each candidate, operator can:
   - **Approve & Refresh** — Claude rewrites the post in-place, it goes live immediately
   - **Skip 30 days** — defers the candidate; it re-appears in next month's queue
   - **Exclude Forever** — sets `refreshExcluded: true` in Sanity; never appears again

### Refresh Execution

When approved:
1. Full post fetched from Sanity (body, title, excerpt, meta)
2. For `fast-changing`/`competitive` + `full-refresh`: 1 Tavily search for fresh context
3. Claude rewrites: title, excerpt, body, metaTitle, metaDescription
4. Sanity doc patched in-place: `lastRefreshedAt = now`, `refreshCount++`
5. Post stays `workflowStatus: 'published'` — live immediately via ISR

### Configuration

All thresholds live in `lib/refresh-config.ts`. To tune:
- **Minimum age** — change `REFRESH_CONFIG.minAgeDays` (default: 90)
- **Cadences** — update `REFRESH_CONFIG.cadences[tier]`
- **Scoring weights** — update `REFRESH_CONFIG.scoring`
- **Never-refresh posts** — add slugs to `REFRESH_CONFIG.excludedSlugs` or set `refreshExcluded: true` in Sanity Studio

### GA4 Extension Point

Traffic-based scoring is stubbed and ready to wire up. `computeScore()` in `lib/refresh-engine.ts` accepts an optional `ga4Data?: Map<string, GA4PostData>`. When wired: declining sessions, striking-distance rankings (positions 4–15), and high-value declining posts add up to +30 additional points. See `GA4PostData` interface in `lib/refresh-engine.ts`.

## Video Pipeline (HeyGen)

### How it works

HeyGen generates an AI avatar video from a script. The VA uses the generated "base video" as raw footage — they download it, edit it externally (add B-roll, background music, subtitles, etc.), then upload the final cut back through the VA editor for distribution.

### Steps

1. VA opens a post in the VA editor
2. Video Script panel shows a pre-generated script (Claude-written, editable)
3. VA enters their HeyGen **Avatar ID** and **Voice ID** (from HeyGen dashboard)
4. Clicks **Generate Base Video** → `/api/content/generate-heygen-video`
5. HeyGen generates the video (1–5 minutes depending on length)
6. Status polling via `/api/content/heygen-status` every 5s — shows progress message
7. When complete: the generated video is saved to Vercel Blob and a **Download Base Video** link appears
8. VA downloads, edits in their video editor, exports final MP4
9. VA returns to the VA editor → **Step 2: Upload Final Video**
10. VA uploads the final MP4 → stored in Vercel Blob with random suffix (no duplicate errors)
11. VA optionally uploads a video thumbnail image
12. After the post is published to website/Facebook, clicks **▶ Publish Video to YouTube & TikTok**

## Market & Research Priorities

**ALL articles must be about Virginia Beach, Chesapeake, Norfolk, Suffolk, Hampton Roads, or directly relevant Virginia law/policy.** Articles about other states score ≤1 and are dropped.

High-value topic weights in the scoring model:

1. **Hampton Roads property values & investment** — home price forecasts, rental returns, neighborhood comparisons
2. **Virginia law changes** — property tax, HOA regulations, zoning, tenant/landlord, real estate transactions
3. **Major development projects** — Port of Virginia, resort area, downtown arena, corporate relocations
4. **Military sector** — base expansion, PCS season, VA loans, BAH rates, NAS Oceana
5. **Flood & coastal risk** — FEMA flood zones, flood insurance, sea level rise (Norfolk is one of the fastest-sinking US cities)
6. **Cost breakdowns** — closing costs, HOA fees, property taxes, insurance — Renick's #1 format by engagement lift

## Enhanced Category Strategy

*Based on Renick Blog Effectiveness Dashboard analysis — April 2026*

**`cost-breakdown`** — Every post includes a summary table in the first third. Renick's #1 format (5 of his top 10 posts).

**`flood-and-risk`** — Flood zone classification, insurance costs, sea level rise. Renick's single highest-lift post was an insurance explainer (+1829%). Currently underrepresented.

### Format Rules (All Categories)

1. **Question-led titles by default.** "What Does It Cost…", "How Do… Work?", "Is X or Y Better?" Only use statement titles when question form is awkward.
2. **Include a real number in the title** whenever possible.
3. **Cost breakdown posts:** Summary table required, placed in the first third.
4. **Comparison posts:** Must end with an explicit "Best For…" conclusion.
5. **Process guides:** Numbered steps, not prose. Each step: what happens, who does it, how long.
6. **All market posts:** "What This Means For Hampton Roads Buyers/Sellers" is non-negotiable.
7. **Military-angle posts:** Specify active duty vs. veterans vs. both. BAH rates, VA loan entitlement, and PCS timelines vary.

## Adapting This Pipeline for a New Client

1. **New Sanity project** — create project, update `NEXT_PUBLIC_SANITY_PROJECT_ID` and `NEXT_PUBLIC_SANITY_DATASET`
2. **Market geography** — update pinned queries and rotating pool in `lib/research.ts` to match the client's metro area
3. **Agent voice** — update the system prompt in `lib/writer.ts` with the agent's name, years of experience, local anchors, and specialty
4. **Social copy persona** — update the Claude Haiku prompt in `lib/publish-service.ts` → `generateSocialCopy()`
5. **Blotato credentials** — new `BLOTATO_API`, `BLOTATO_FACEBOOK_ACCOUNT_ID`, `BLOTATO_FACEBOOK_PAGE_ID`, `BLOTATO_YOUTUBE_ACCOUNT_ID`, `BLOTATO_TIKTOK_ACCOUNT_ID` for the client's social accounts
6. **Community photos** — add `public/community-photos/[city]/` assets for the new market + update the static manifest in `lib/thumbnail-asset-resolver.ts`
7. **Agent photo** — replace `public/barry-transparent.png` with the new agent's transparent PNG
8. **Production URL** — update `NEXT_PUBLIC_APP_URL` in Vercel env vars
9. **Emails** — update `OPERATOR_EMAIL` and `BARRY_EMAIL` in Vercel env vars
10. **Blog post categories** — verify the category list in `sanity/schema/blogPost.ts` matches the client's content strategy; add/remove as needed
11. **YLOPO widget** — update `detectCommunities()` in `app/(site)/blog/[slug]/page.tsx` to match the new market's city names
12. **HeyGen avatar** — configure avatar ID and voice ID for the client's agent; VA enters these in the VA editor per post

**Phase 2 multi-client (future):** A Sanity `clientConfig` document will store Blotato credentials, agent photo, Facebook/YouTube/TikTok account IDs, and market config per client. `lib/blotato-client.ts` is designed so `getAccountId()` / `getPageId()` can be swapped from env vars to Sanity config without changing the publish call.
